# 家财管家 MVP 需求规格

## 目标

把家庭的账户余额、实际流水、内部资产转换、固定资产、负债、期权、未来资金安排与证照期限放在一个可勾稽的本地账本中。首版优先回答四个问题：现在可动用多少钱、家庭净资产是多少、未来资金何时紧张、哪些合同或证照需要处理。

## 非目标

- 不直连银行、券商或保险机构；首版通过手工录入和安全导入草稿更新。
- 不存储密码、PIN、CVV、验证码、恢复码、助记词或完整账号/证件号。
- 不把资产转换误报为收入或支出，不把市场估值当作已实现现金流。

## 业务规则

1. 账户余额是带日期的快照；资产与负债使用独立估值/余额记录。
2. 收入和支出影响一个账户；内部转账必须同时有来源与去向，并在家庭口径净额为零。
3. 预测现金流不直接改变账户余额。发生后必须登记实际流水，再在预测中标记已发生或取消。
4. 净资产 = 金融账户余额 + 按权益比例计算的资产估值 − 按权益比例计算的关联负债。
5. 每笔外币记录保存原币和汇率来源；首版界面暂以单币种展示，跨币种汇总须人工确认。
6. 证照、合同和保险仅存名称、归属、有效期、状态和外部保险库引用。

## MVP 模块

- 总览：净资产、可动用资金、90 天预计流出、行动清单和流动性结构。
- 账户：账户台账、最新余额、归属、流动性和快照日期。
- 流水：收入、支出、内部转账；转账自动同时更新两个账户。
- 资产负债：资产类别、权益比例、估值日期、毛值、关联负债和净值。
- 资金预测：基准/保守情景、预计流入流出、计划与实际分离。
- 证照提醒：证照、合同、保险的到期日和外部保险库引用。
- 数据安全：本地导入、导出备份、恢复脱敏示例与敏感信息边界。

## 月结流程

1. 新增或更新账户余额快照。
2. 导入或录入本月实际流水；配对所有内部转账。
3. 将已发生的预计现金流标记为已发生，并解释计划与实际差异。
4. 更新房产、证券、期权和负债的估值日期及来源。
5. 检查 90 天内资金缺口、到期证照、保险、存款和期权窗口。
6. 导出加密位置保存的本地备份。

## 验收标准

- 可新增账户、收入、支出、内部转账、资产、预计现金流和证照记录。
- 内部转账后来源账户减少、去向账户增加，金融资产总额不变。
- 总览随记录变化重新计算净资产、流动性与提醒数量。
- 预测不修改实际余额；实际流水不自动覆盖预测。
- 支持 JSON 导出和恢复导入。
- 界面与导入器不出现完整账号、证件号、密码、PIN、CVV 或验证码字段。
